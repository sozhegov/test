Задания необязательно выполнять в полном объеме. Главное - чтобы можно было адекватно оценить Ваши проф. навыки.

1. Набросать класс(ы), свойства и методы для расчета Розничной цены конкретного товара.

a. Есть рекомендованные цены мелких поставщиков на товар с количеством этого товара у каждого поставщика
b. Есть рекомендованные цены крупных поставщиков на товар (удаленные склады) с количеством этого товара 
у каждого поставщика
c. Есть некие расчетные(наши, минимально возможные) цены крупных поставщиков на товар с количеством этого 
товара у каждого поставщика

Кол-во товара b и c совпадает (b и c можно реализовать массивом(-объектом) 1й и той же сущности, на Ваше усмотрение). 

Есть наш основной склад, куда поступает товар от поставщиков, коллекционеров и просто частных лиц:
- с ценой покупки (Входная цена - по которой купили у поставщика или частного лица) 
- с ценой продажи (Выходная цена - по которой продаем, не может быть меньше Входная*минНаценка).
- кол-во товара

Исходя из этих параметров сделать вывод расчетной цены на данный товар, которая будет показана покупателю при условиях:

1. Если товар есть на НАШЕМ складе, то берем минимальную цену только из a и b, но больше чем Входная*минНаценка.
Иначе выводим Входная*минНаценка.

2. Если товара нет на нашем складе, берем минимальную цену из c.

3. Если у товара нет никаких данных по поставщикам (a,b,c пустые), выводим Выходную цену 
(на всякий, сделать доп. проверку, что она выше или равна Входная*минНаценка).

----------

2. Набросать алгоритм расчета (метод(ы)) Выходной цены на товар у поставщиков типа c (типы выше).

К 1му товару может быть привязано несколько поставщиков.
У каждого поставщика есть цена на данный товар, по которой мы у него его купим - Входная цена.

Для каждого поставщика рассчитать Выходную цену (=Входная*Наценка), по которой мы сможем продавать этот товар, 
купив его у данного поставщика, исходя из условий:

1. (ЭТОТ ПУНКТ ОБЯЗАТЕЛЕН К ВЫПОЛНЕНИЮ КАНДИДАТОМ) Для каждого поставщика прописан массив наценок в зависимости от Входной цены, например: 
45% - для Входной цены от 0 по 499
40% - от 500 по 999
35% - от 1000 по 4999
и т.д.

2. Для поставщика может быть указана также отдельная наценка для товаров, 
у которых указан параметр - Скидок нет, например:
50% - для всех товаров, у которых discountless==true

3. Если наценка не указана, то она 25%.

----------

3. Набросать запрос(ы) (чистый SQL или с помощью билдера yii Query).

есть таблица товаров, в ней есть привязка к производителю
есть таблица привязок товаров к категориям (товар может относиться к нескольким категориям)
есть таблица привязок к доп полям (к примеру, есть табл с масштабами автомоделей. 
У товара соотв. только 1 масштаб)

Написать запрос(ы) фильтра по производителям, категориям, масштабам (сужение выдачи, 
если выбраны разные типы фильтров - AND). 
Можно одновременно выбирать по-нескольким производителям, категориям или масштабам (расширение выдачи - OR). 
При желании для наглядности кода его можно разделить на методы в классе фильтра.
