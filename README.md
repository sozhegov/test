Судя по всему сайт построен на YII2.

Создадим несколько сущностей(таблиц):

Product (товар)
Поля:
1. pid (id товара)
2. name (название товара)
3. qty (количество на нашем складе)
4. inPrice (входная цена на нашем складе)
5. outPrice (выходная цена на нашем складе)
6. discountLess (0/1)
    
Vendor (поставщик)
Поля:
1. vid (id постащика)
2. name
3. discountLessCharge (наценка для товаров discountLess=1)
4. defaultCharge (дефолтная наценка, если цена не попадает ни в один диапазон из сущности PriceCharge)
5. type (a/b/c)

Stocks (складские остатки)
Поля:
1. vid (id постащика)
2. pid (id товара)
3. price
4. qty (остаток на складе)

PriceCharge (множитель цен для поставщиков)
Поля:
1. vid (id постащика)
2. fromPrice (минимальная цена диапазона)
3. toPrice (максимальная цена диапазона)
4. charge (множитель, например 1.2 = наценка 20%)
5. chargeFound (всегда = 1 - позволяет понять найден диапазон или нет)

Categories (категории)
1. cid
2. parentCid
3. name

ProductCategory (привязка категорий к товарам)
1. cid
2. pid

-----------------------------------------------------------

Создадим класс Product и добавим в него методы:
1. getChargedProductPrices($pid) - этот метод вытаскивает из БД минимальные цены для каждой категории (a, b, c)

Q1: select ... from products as p left join stocks as s on s.pid=p.pid where pid={$pid} - эта часть запроса отсечет список vid у которых есть товар
Q2: select ... from Q1 as q1 left join price_charge as pc on pc.vid=q1.vid where q1.price >= pc.fromPrice and q1.price <= pc.toPrice - эта часть запроса оставит только те стоки, которые проходят по диапазону PriceCharge
Q3: select ... from Q2 as q2 left join vendor as v on v.vid=q2.vid - тут мы приклеиваем данные о множителе для discountLess=1 и defaultCharge

В результате по каждому поставщику образуется строка с 4 ценами: 1 - наша цена(outPrice), 2 - цена для товаров discountLess=1, 3 - цена для defaultCharge, 4 - цена товара при попадании в диапазон (rangePrice)
Цена товара при попадании в диапазон может быть пустой.

В итоге цена товара для вендора будет определяться по формуле vendorPrice = (discountLessPrice * discountLess + (defaultCharge * (1 - chargeFound) + rangePrice * chargeFound) * (1 - discountLess))

Теперь мы можем отсортировать поставщиков по типу (a,b,c) и vendorPrice, если мы хотим получить цену на товар у всех поставщиков или можем сгруппировать по типу поставщика и получить минимальную цену в каждой группе для расчета цены, которую нужно показать покупателю.
Убрав из Q1 фильтрацию по PID можно построить запрос, который выдаст цены по типу поставщика(a,b,c) сразу на все товары.

P.S.Вообще запрос довольно сложный и ресурсоемкий и был придуман скорее в демонстрационных целях. Если товаров на сайте много, то лучше кэшировать его результаты и загонять в отдельную таблицу или поисковый индекс на этапе обновления остатков чтобы пользователи могли фильтровать товары по цене.
