| `pid`          | id товара                     |

| `inPrice`      | входная цена на нашем складе  |
| `outPrice`     | выходная цена на нашем складе |
| `discountLess` | 0/1                           |

### Vendor (поставщик)

| Поле                 | Описание                                                                     |
|----------------------|------------------------------------------------------------------------------|
| `vid`                | id поставщика                                                                |
| `name`               | —                                                                            |
| `discountLessCharge` | наценка для товаров `discountLess=1`                                         |
| `defaultCharge`      | дефолтная наценка, если цена не попадает ни в один диапазон из `PriceCharge` |

### Stocks (складские остатки)

| Поле    | Описание          |

| `price` | цена              |

### PriceCharge (множитель цен для поставщиков)

| Поле          | Описание                                                 |
|---------------|----------------------------------------------------------|
| `vid`         | id поставщика                                            |

| `charge`      | множитель, например `1.2` = наценка 20%                  |
| `chargeFound` | всегда `= 1` — позволяет понять, найден диапазон или нет |

## Класс `Product`

Создадим класс `Product` и добавим в него методы.

### 1. `getChargedProductPrices($pid)`

Метод вытаскивает из БД минимальные цены для каждой категории (a, b, c).

```sql
-- Q1: отсекает список vid, у которых есть товар
select ... from products as p
left join stocks as s on s.pid = p.pid
where pid = {$pid}

-- Q2: оставляет только те стоки, которые проходят по диапазону PriceCharge
select ... from Q1 as q1
left join price_charge as pc on pc.vid = q1.vid
where q1.price >= pc.fromPrice and q1.price <= pc.toPrice

-- Q3: приклеивает данные о множителе для discountLess=1 и defaultCharge
select ... from Q2 as q2
left join vendor as v on v.vid = q2.vid
```

В результате по каждому поставщику образуется строка с 4 ценами:

1. наша цена (`outPrice`);
2. цена для товаров `discountLess=1`;
3. цена для `defaultCharge`;
4. цена товара при попадании в диапазон (`rangePrice`).

Цена товара при попадании в диапазон может быть пустой.

В итоге цена товара для вендора определяется по формуле:

```
vendorPrice = discountLessPrice * discountLess
            + (defaultCharge * (1 - chargeFound) + rangePrice * chargeFound) * (1 - discountLess)
```

Теперь мы можем:

- отсортировать поставщиков по типу (a, b, c) и `vendorPrice`, если хотим получить цену на товар у всех поставщиков;
- либо сгруппировать по типу поставщика и получить минимальную цену в каждой группе — для расчёта цены, которую нужно показать покупателю.

Убрав из Q1 фильтрацию по `pid`, можно построить запрос, который выдаст цены по типу поставщика (a, b, c) сразу на все товары.

> **P.S.** Вообще запрос довольно сложный и ресурсоёмкий и был придуман скорее в демонстрационных целях. Если товаров на сайте много, то лучше кэшировать его результаты и загонять в отдельную таблицу или поисковый индекс на этапе обновления остатков, чтобы пользователи могли фильтровать товары по цене.
